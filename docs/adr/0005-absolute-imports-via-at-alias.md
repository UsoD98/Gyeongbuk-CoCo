# 0005. 절대 경로 `@/` 별칭 강제

- 상태: Accepted
- 결정일: 2026-05-02
- 결정자: 프런트엔드 팀

## 맥락

- 상대 경로(`../../../components/...`)는 파일 이동 시 잘 깨지고 가독성이 나쁘다.
- 컴포넌트가 어느 레이어에 있는지 import 한 줄로 즉시 보이게 하고 싶다.

## 고려한 대안

- **상대 경로 유지** — 별도 설정 없음. 단점: 깊은 경로일수록 가독성 저하, 리팩터링 비용.
- **여러 별칭(`@components/...`, `@stores/...`)** — 시멘틱하지만 별칭 추가/관리가 늘어남.
- **단일 `@/` 별칭 = `src/`** — Vite·TS·ESLint 어디서나 한 줄 설정. 폴더 의미는 경로 그 자체로 표현(`@/stores/...`).

## 결정

**`@/` 단일 별칭 = `src/`** 채택. 앱 코드 import는 모두 `@/`로 시작한다. 상대 경로 금지.

설정 위치:
- `vite.config.ts`의 `resolve.alias['@']`
- `tsconfig.app.json`의 `compilerOptions.paths['@/*']`

## 결과

긍정적:
- 파일 이동 시 import 손상이 없다.
- 컴포넌트가 어느 도메인·레이어에 속하는지 import 한 줄로 보인다.

부정적:
- 별칭을 모르는 외부 도구(특정 IDE 플러그인 등)가 가끔 경고할 수 있음. tsconfig·vite 양쪽이 동기화되어 있으면 실 사용엔 문제 없음.

후속:
- 규약 강제는 [CONVENTION.md §2](../../CONVENTION.md) "임포트" 섹션에 명시. 코드 리뷰에서 상대 경로는 차단.
