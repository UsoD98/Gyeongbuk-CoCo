# 0010. 인증 게이팅 정책 — 저장·공유에 로그인 필수 여부

- 상태: Accepted
- 제안일: 2026-05-30
- 결정일: 2026-05-30
- 결정자: 프런트엔드 팀

> 출처: [PRD §10 OQ6](../PRD.md#10-가정-및-미결-open-questions).

## 맥락

- 카카오 OAuth 위젯은 이미 존재한다(`src/components/auth/KakoLoginComponent.tsx`, [ADR 0006](./0006-kakao-login-via-react-kakao-login.md)).
- 어떤 기능에 로그인을 요구할지 정책이 없다 — 특히 **컬렉션 저장**과 **코스 공유**.
- 제안서의 단기 발전 방향([PRD §8](../PRD.md#8-발전-방향--기대효과))은 UGC(사용자 작성 코스·실지출) 축적을 전제하므로 계정 귀속이 유리하다.

## 고려한 대안

- **A: 저장·공유 시 로그인 필수** — 데이터 소유·UGC 축적·기기 간 동기화에 유리. 단 진입 마찰 증가.
- **B: 게스트 허용(로컬 저장) + 익명 공유** — 마찰 최소. 단 기기 간 동기화·UGC 귀속 불가.
- **C: 하이브리드** — 탐색·플래닝은 게스트, **저장·공유 시점에 로그인 유도**.

## 결정

**C(하이브리드) 채택.**

- **탐색·검색·플래닝(코스 빌더·예산 계산)은 게스트로 자유롭게** 사용한다 — 진입 마찰 최소화(가성비·간편함 가치).
- **저장(컬렉션)·공유 시점에만 로그인을 유도**한다. 로그인하지 않으면 해당 액션 직전에 카카오 로그인 모달/리다이렉트로 안내한다.
- 인증 상태는 신설 `authStore`(Zustand)로 관리한다. UI-우선 단계에서는 더미/로컬 로그인 상태로 동작하고, 백엔드 연동 시 실제 세션으로 교체한다.

## 결과

긍정적:
- 핵심 가치(탐색·플래닝)는 무마찰, 데이터 귀속이 필요한 시점에만 인증 → UGC 축적과 간편함을 절충.

부정적/후속:
- 라우트 전체 가드가 아니라 **액션 단위 게이트**라 저장·공유 호출 지점마다 가드 호출을 일관되게 끼워야 한다(헬퍼/HOC 또는 훅으로 캡슐화).
- 컬렉션 저장은 현재 localStorage 지속(themeStore 패턴), 추후 서버 동기화로 교체 → 게스트 로컬 저장분과 로그인 후 서버 저장분의 병합 정책을 백엔드 연동 시 정한다. → [백엔드 협의 문서](../backend-api-contract.md)
- `authStore` 신설(현재 `themeStore`만 존재).
